Council Opens the Waterfront Ledger After Midnight Vote

A dense broadsheet system for credible publishing: serif mastheads, ruled datelines, tight decks, justified columns, pull quotes, gray photographs, and one broadsheet red for real emphasis.

BY MARA CHEN · HARBOR DISTRICT · FILED 6:20 A.M.

Harbor residents woke to a freshly printed plan for the east pier after council members voted through a public-access agreement just before midnight. The document adds ferry sheds, market stalls, flood gates, and a narrow promenade to a site that has been fenced for nearly a decade.

The front page is intentionally compact. Headlines carry the first read, decks carry the second, and the body settles into measured columns divided by hairline rules. Every separator is ink, not shadow.

“A page can be crowded and still be orderly when the rules are honest.”

Reporters at The Harbor Ledger use small-caps kickers to signal sections, mono datelines to fix time and edition, and a single red to mark urgent broadsheet decisions. More color would make the page feel like a poster instead of a paper.

On narrow screens the columns collapse to one column, the masthead compresses, and the table demos remain horizontally scrollable. The visual language stays print-like without forcing a desktop sheet onto a phone.

午夜后的码头表决：一份港湾日报的版面记录

BY LIN MU · HARBOR DISTRICT · A1 · 2026 年 6 月 18 日

这篇示例展示报纸正文如何使用 kicker、deck、byline、首字下沉、rule、quote 和多级标题，同时仍然消费固定 --ds-* token。

午夜刚过，港湾议会通过了东码头开放草案。报社的 A1 版面在二十分钟内改稿：主标题压到两行，deck 保留三十余字，旁栏塞进三条背景，图片改成灰阶并加上黑色 keyline。

Newspaper kit 的目标不是放松阅读，而是高密度扫描。读者先看 masthead 确认刊物，再看 kicker 判断版面，随后由 headline、deck 和 byline 进入正文。规则线承担节奏，红色只承担强调。

报头，是第一层可信度

大号 nameplate、双线 rule、dateline、edition、vol/no 与 price 共同完成身份声明。它们不是装饰，而是“这是一份报纸”的第一眼证据。

“A crowded page is readable when every rule has a job.”

— The Harbor Ledger copy desk

新闻纸，而非冷白

亮色主题使用 --ds-bg: #f2efe8 与 --ds-fg: #1a1a18，模拟油墨落在新闻纸上的高对比。夜刊主题反相为深墨底与浅纸字，并把编辑红提亮到 #e0564e。

把强调留给真正重要的事

整套系统只有一种强调色：编辑红。它出现在 kicker、焦点环、危险状态和少量重点标记中。其余的一切，都交给墨、纸、字重和 rule。

/* 报纸正文契约：衬线 + 多栏 + 栏线 */
.story-columns {
  font-family: var(--ds-font-serif);
  font-size:   var(--ds-text-base);   /* 17px */
  line-height: var(--ds-leading-relaxed); /* 1.85 */
  column-count: 3;
  column-rule: 1px solid var(--ds-border);
}

最后，关于响应式。桌面可以使用三栏和旁栏；手机必须收成一栏，headline 降档，表格横滑。报纸味来自层级和规则，不来自让窄屏横向滚动。

  • 正文、标题、masthead 全部使用 serif，控件和表单才使用 sans。
  • 用 column-rule、double rule 和 boxed quote 建立报纸层级。
  • 移动端把多栏收成一栏，避免 390px 宽度横向溢出。

报纸图片统一灰阶、轻微加对比，并用 keyline 收边；caption 一律无衬线 muted。素材取自本地共享图库，全部 loading=lazy + 有意义 alt。

灰阶图片与正文并排，形成主版第二入口

图文混排时，照片、headline、deck 与 caption 必须共享同一版面节奏。图片不做彩色营销感，而是像报纸图片一样收进墨色系统。

正文走 serif，元信息与说明走 sans/mono；所有边线和底色继续消费 --ds-* token。

Column System

反向布局，让报纸长页保持节奏

同一组件交替左右图位，长页面便有了起伏。每一屏保持明确 rule、kicker 和 deck，不让密度变成混乱。

图注用 --ds-fg-muted，与正文拉开但仍达 AA；图片可在容器内对齐，也可出血到更宽的布局区。
