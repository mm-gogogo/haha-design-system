# Gov-CN · 中国政府风（政务）

> 庄重、权威、规整、可信，严肃但不压抑。给政务门户、舆情系统、文旅政务、公共服务平台用的设计系统。视觉惯例参考中国政府网与各地政务门户：**红色横幅 + 标题栏、栏目清晰、通知公告条目列表（日期右对齐）、规整表格、稳重按钮（红主蓝次）**。

---

## 1. 这套风格是谁、给谁、什么感觉

中国政府风是一套面向**官方/公共服务**场景的设计系统。它的目标不是"好玩"或"个性"，而是让用户在第一眼就建立**权威感与信任感**：这是政府/官方机构发布的、严肃、可靠的信息。

- **谁**：政务门户首页、政务公开、办事大厅、舆情监测后台、文旅政务、12345 热线、公共数据平台。
- **感觉**：端正、对称、清晰、克制。像一份盖了红章的正式公文，而不是一款消费 App。

权威感的三个来源：
1. **党政红横幅**——顶部一条红色 banner + 标题栏，是中国政务门户最强的视觉符号。
2. **规整对称的版式**——栏目分明、信息列表密集、日期右对齐的"通知公告"气质。
3. **方正的形态语言**——小圆角、克制阴影、靠描边与分隔线表达层级。

---

## 2. 设计哲学 / 性格

继承 Geist 的**语义分层 + 比例尺 + 状态色 + 焦点环**骨架，但在气质上做了政务化改造：

1. **庄重优先**：圆角小而方正（2/4/6px），不圆润；阴影极克制，层级主要靠**描边与分隔线**——像公文表格那样规整。
2. **双主色秩序**：**党政红 `--ds-primary`** 用于横幅、主操作、强调；**政务蓝 `--ds-accent`** 用于链接、次操作、信息态。红主蓝次，分工明确，绝不混用。
3. **暖中性纸感**：次背景用暖灰 `#f5f3f0`（纸感），而非冷灰或纯白堆叠，呼应公文纸张的质感，缓解红色的攻击性。
4. **荣誉金点缀**：`--ds-gold #c9a227` 仅用于荣誉/徽标/证书的**描边或细节**，绝不大面积铺底。
5. **信息密集但有呼吸**：通知列表、表格要密，但用分隔线和稳定的行高保持可读，做到"严肃但不压抑"。

与 Geist 的关系：**继承**语义 token 名、比例尺思路、双层焦点环、明暗双主题；**改**——中性色换成暖纸感、强调色换成红蓝双主色、圆角变小、阴影变弱、字体换成黑体正文 + 宋体标题。

---

## 3. 何时用 / 何时别用

**适合：**
- 政府/事业单位/官方机构的门户、公开平台、办事系统。
- 需要强权威感与公信力的公共服务、舆情、政务数据后台。
- 文旅政务、公共通知发布、政策解读类信息站点。

**不适合：**
- 消费类、娱乐、潮流品牌——红色横幅与公文气质会显得过于严肃。
- 强调个性、活泼、年轻化的产品（用 cute / pixel / editorial 等风格）。
- 需要大量沉浸式图片/视频的营销站（政务风重文字、轻图饰）。

## 4. 颜色

红主蓝次。红色承载横幅与主操作，蓝色承载链接与信息，暖灰承载背景纸感。状态色取稳重的中明度，避免荧光感。**注意：红底白字对比需特别校验**（见可访问性）。

## 5. 字体

权威感来自字体的"端正"。

- **`--ds-font-sans`（正文/UI）**：拉丁用 `"Inter"`，中文用 `"Source Han Sans SC"`（思源黑体）优先，回退 `Noto Sans SC / PingFang SC / Microsoft YaHei`。黑体清晰、中性、严肃，适合密集的通知列表与表格。
- **`--ds-font-serif`（标题/权威）**：拉丁用 `"Noto Serif"`，中文用 `"Source Han Serif SC"`（思源宋体）优先，回退 `Songti SC / SimSun`。宋体的横细竖粗、笔锋分明，最能体现公文与碑刻式的权威感——**横幅大标题、栏目标题建议用 serif**。
- **`--ds-font-mono`（数据）**：`"Source Han Mono SC"` 回退系统等宽，用于编号、统计数字。

**比例尺**（与契约一致）：xs 12 / sm 13 / base 14 / md 16 / lg 20 / xl 24 / 2xl 32 / 3xl 48。
**字重**：normal 400 / medium 500 / **bold 700**（标题用 700，比 Geist 的 600 更重，强化庄重感）。
**行高**：tight 1.25 / normal 1.6 / relaxed 1.9——比 Geist 略松，中文密集排版更易读。
**字距**：`0`（中文不做负字距，保持端正）。

**真实字体嵌入**：`preview.html` 在 `<head>` 顶部引入 `../../_fonts/fonts.css`（仓库共享、本地 woff2、OFL 开源，唯一允许的样式外链，不引任何外网 CDN）。**拉丁字形**走嵌入字体——sans 以 `"Inter"` 打头、serif 以 `"Noto Serif"` 打头；**中文**仍沿用系统回退：正文黑体 `Source Han Sans / PingFang`，标题宋体 `Source Han Serif / Songti`。因此英文/数字端正现代，中文权威端庄，无需额外中文字体文件。

**中文适配**：所有字族在拉丁字体之后即以 SC 简体中文字体续接并提供系统回退（`Source Han / Noto SC / PingFang / Songti / SimSun`）；缺字体的环境也能离线显示。

## 6. 间距与布局

- **比例尺（4px 基）**：1–12 映射 4,8,12,16,20,24,32,40,48,64,80,96 px。
- **节奏**：列表条目内 8–12px、条目间用分隔线（0 间距 + border）、栏目区块间 32–40px、页面区块间 48–64px。
- **容器宽**：政务门户建议主容器 `1200px` 居中（参考 Geist xl 断点），内容区与侧栏二八/七三分。
- **栅格**：规整对称 12 栏。首页常见"要闻轮播 + 通知公告 + 政务服务入口"的三段式对称布局。
- **关键气质**：**对称**。左右栏宽度、上下间距尽量成对、成倍，避免随手取值——对称本身就是权威感。

### 响应式 / 移动端

政务门户需兼顾桌面与"掌上办事"。`preview.html` 采用单断点策略，规整可预期：

- **断点**：`≤860px` 为平板过渡（三/四列栅格降为两列）；**`≤640px` 为手机端**——`.grid-2/3/4` 一律收一列；红色 banner 由横排转纵排（站名缩为 `xl`、徽标缩小、主题按钮右对齐）；标题栏主导航折叠为**可横向滚动的单行**（移动惯例），面包屑同样横向滚动避免折行；正文左右留白收窄到 `space-4`。
- **手机框示例**：新增"移动端 Mobile"块，约 `375px` 宽手机框复用同一套 `var(--ds-*)`，自上而下为——**红头标题栏**（`--ds-primary` 底 + `--ds-gold` 金线，与桌面 banner 同语言）、**面包屑**、**办事入口宫格**（4 列触控目标，图标用语义浅底）、**通知公告列表**（分隔线 + 日期右对齐）。
- **触控**：移动端入口与列表项目标 ≥ 44px；横向滚动区保留 `-webkit-overflow-scrolling: touch`。
- **原则**：移动端不改 token、不改组件结构，只靠栅格收列与容器内边距收窄；红底白字、分隔线层级、方正圆角在小屏下完全保持，权威气质不退化。

---

## 7. 圆角 / 阴影 / 描边

- **圆角**：sm `2px` / md `4px` / lg `6px` / full `9999px`。**小而方正**是政务风的核心形态语言，按钮、卡片、输入框都保持低圆角，庄重不圆润。
- **阴影**：极克制。`--ds-shadow-sm` 几乎只是一条底线，`md/lg` 也很浅。**层级优先用描边与分隔线表达**，而非投影——这呼应公文表格的视觉惯例。
- **描边**：`--ds-border #e0ddd6`（常规）/ `--ds-border-strong #c8c4ba`（分隔、表头）。横幅与卡片可叠加 `--ds-gold` 细描边表达"荣誉/重要"。分隔线是本风格表达结构的主力。

---

## 8. 动效

- **时长**：fast `150ms` / base `200ms` / slow `250ms`——全部在 150–250ms 区间，极克制。
- **曲线**：`--ds-ease: cubic-bezier(.4,0,.2,1)`（标准 ease），稳重无弹跳。
- **典型过渡**：按钮 hover/active 的底色渐变、tab 切换的下划线移动、列表 hover 的底色淡入。**不做花哨动画**（无缩放弹跳、无旋转、无粒子）。
- **`prefers-reduced-motion`**：必须支持。在 reduce 下移除所有过渡（`transition: none`），仅保留即时状态变化。

## 9. 组件规格

### 按钮（全状态）
| 类型 | 底色 | 文字 | 边框 | 高 | 圆角 |
|---|---|---|---|---|---|
| primary（主·红） | `--ds-primary` | `--ds-primary-fg` | 无 | 40px | `sm` 2px |
| secondary（次·蓝描边） | `--ds-bg` | `--ds-accent` | 1px `--ds-accent` | 40px | `sm` |
| tertiary（文本） | 透明 | `--ds-accent` | 无 | 40px | `sm` |
| danger | `--ds-danger` | `#fff` | 无 | 40px | `sm` |

- 尺寸：小 32 / 中 40 / 大 48px；内边距 `0 16px`。
- 状态：hover 用 `--ds-primary-hover`；active 用 `--ds-primary-active`；disabled 降不透明度至 .5 且 `cursor:not-allowed`；focus 加 `--ds-focus-ring`。

### 输入框 / 表单
- input/select/textarea：底 `--ds-bg`（或 sunken）、边 1px `--ds-border`、高 40px、内边距 `0 12px`、圆角 `sm`。
- focus：边变 `--ds-accent` + `--ds-focus-ring`。
- checkbox/radio/switch：选中态用 `--ds-primary`（红）；radio 圆，checkbox/switch 方正。

### 卡片
- 底 `--ds-bg-elevated`、边 1px `--ds-border`、圆角 `md` 4px、内边距 `--ds-space-6`、阴影 `sm`（几乎无）。栏目卡顶部常配红色或金色细条作标识。

### 徽章 / 标签
- 端正方正，圆角 `sm` 2px，内边距 `2px 8px`，字号 `xs`。各语义色用对应 `-bg` 浅底 + `-fg` 深字（如 info：`info-bg` 底 + `info-fg` 字）。荣誉徽标用 `gold` 描边。

### 提示条 alert
- 四类：success / warning / danger / info。结构：前置同色语义内联 SVG 图标（对勾 / 感叹三角 / 圆叹号 / 圆 i）+ 浅底（`*-bg`）+ 1px 同色全描边 + 深字（`*-fg`）+ 同色标题。圆角 `md`。不用侧边色条。

### 导航
- **顶部红色 banner**：`--ds-primary` 满宽底 + 白字站名 + serif 大标题，可叠 `gold` 细底线。
- **标题栏 / 面包屑**：banner 下方一条标题栏，面包屑用 `--ds-fg-muted`，分隔符用 `/`，当前页 `--ds-fg`。
- **tab / 栏目导航**：当前项底部 2px `--ds-primary` 下划线 + 加粗；hover 底 `--ds-bg-subtle`。

### 面包屑 Breadcrumb（政务必备）
- 结构：`首页`（前置房屋内联 SVG 图标，`--ds-primary`）→ 分隔符 `/`（`--ds-fg-subtle`）→ 各级链接（`--ds-accent`，hover 底 `--ds-info-bg`）→ 当前页 `aria-current="page"`（`--ds-fg` 加粗、不可点）。
- 政务路径常较深（首页/政务服务/个人办事/具体事项），移动端可横向滚动避免折行。

### 步进器 / 办事流程步骤条 Stepper（在线办事）
- 横向五节点：圆形序号 + 标签 + 连接线。三态——
  - **已完成 `.done`**：序号实心红底白字（`--ds-primary`），连接线变红，标签转深。
  - **当前 `.current`**：序号白底红边红字 + 焦点环（`--ds-focus-ring`），标签红色加粗。
  - **待办**：序号白底灰边灰字（`--ds-border-strong`），连接线灰。
- 用于"在线申请 → 材料预审 → 受理审核 → 办结领证 → 评价反馈"这类政务办事流程。

### 下拉菜单 Dropdown
- 触发按钮带下拉箭头内联 SVG（展开时旋转 180°）；菜单 `--ds-bg-elevated` 底 + `--ds-border-strong` 边 + `--ds-shadow-lg`，圆角 `md`。
- 菜单项 hover 底 `--ds-bg-subtle`；**选中项** `.sel` 文字 `--ds-primary` + 行尾对勾内联 SVG；`.divider` 分隔线。点击外部或 Esc 关闭。

### 工具提示 Tooltip
- 触发元素 hover/`focus-within` 显示气泡：深底（`--ds-fg`）浅字（`--ds-bg`）、`xs` 字号、`sm` 圆角、下置小三角。用于解释字段含义、统计口径。键盘可聚焦触发，确保可达。

### 模态框 Modal
- 遮罩 `rgba(26,26,26,.5)` 居中；卡片 `--ds-bg-elevated`、圆角 `lg`、**顶部 4px `--ds-primary` 红条**（政务标识）、`--ds-shadow-lg`。
- 三段式：标题栏（标题 + `×` 关闭）/ 正文 / 操作区（右对齐，次按钮 + 主按钮，底 `--ds-bg-subtle`）。点遮罩或 Esc 关闭，`role="dialog"` + `aria-modal`。

### 分页 Pagination
- 页码块 36px 方正、圆角 `sm`、边 `--ds-border`。hover 转蓝（边/字 `--ds-accent` + 底 `--ds-info-bg`）；**当前页 `.active` 红底白字**（`--ds-primary`）；省略号 `…`；`disabled` 降透明度。可配"共 N 页 · 跳至 [页码]"输入。

### Toast 通知
- 卡片 `--ds-bg-elevated` + 左侧 4px 语义色条（info 蓝 / success 绿 / warning 橙 / danger 红）+ `--ds-shadow-md`。结构：语义图标 + 标题（`--ds-fg`）+ 说明（`--ds-fg-muted`）+ `×` 关闭。用于办理结果、系统提示、材料待补等轻量反馈。

## 10. 可访问性

- **对比度**：正文 ≥ 4.5:1，大字 ≥ 3:1。
  - **红底白字**（`#c4161c` + `#fff`）对比约 **5.9:1**，达 AA，但**禁止在红底上再用小号弱化文字**；按钮文字至少 14px medium。
  - 蓝底白字（`#1c4d8c` + `#fff`）≈ 8.0:1，安全。
  - 暗色下 primary 提亮为 `#e23b41`，其上仍用白字，达标。
- **焦点**：所有可交互元素用双层焦点环 `--ds-focus-ring`（底环 + 政务蓝），键盘可见。
- **触控目标**：≥ 44×44px（按钮默认高 40px，触控场景用大号 48px 或加外边距补足）。
- **可减弱动效**：尊重 `prefers-reduced-motion`，关闭所有过渡。
- **不靠颜色单独传意**：danger 与 primary 同为红，危险操作需配图标/文案/二次确认。

## 11. Do / Don't

**Do**
- 顶部用红色 banner + serif 大标题，立刻建立权威感。
- 通知公告用条目列表，**日期右对齐**，这是政务门户的标志性版式。
- 版式对称、栏目分明、间距成对成倍。
- 层级优先用描边/分隔线，圆角保持小而方正。
- 红主蓝次：红做横幅/主操作，蓝做链接/次操作，分工固定。

**Don't**
- 不要把红色大面积铺满内容区（横幅、按钮够了），否则压抑、刺眼。
- 不要用大圆角、强投影、渐变光效——会失去庄重感。
- 不要红蓝混用主次（如蓝色当主按钮、红色当链接）。
- 不要金色大面积铺底——金仅作描边/徽标点缀。
- 不要堆 emoji 当图标（一律用内联 SVG）；不要硬编码颜色，一律走 `var(--ds-*)`。

## 12. 如何接入

**1) 引入 tokens.css**（提供 `:root` 亮色 + `[data-theme="dark"]/.dark` 暗色变量）：
```html
<link rel="stylesheet" href="tokens.css" />
```

**2) 用变量**（任何 CSS/内联都走变量，切风格零成本）：
```css
.banner   { background: var(--ds-primary); color: var(--ds-primary-fg); }
.btn      { background: var(--ds-primary); border-radius: var(--ds-radius-sm); }
.notice a { color: var(--ds-accent); }
.card     { border: 1px solid var(--ds-border); box-shadow: var(--ds-shadow-sm); }
```

**3) 切换暗色**：
```js
document.documentElement.dataset.theme = 'dark'; // 或移除回到亮色
```

**4) Tailwind 项目**：先在入口引 `tokens.css`，再在 `tailwind.config.js`：
```js
module.exports = { presets: [require('./tailwind.preset.js')] };
// 之后即可用 bg-primary / text-accent / border-strong / rounded-sm / shadow-md / text-gold 等
```

> 变量名严格遵循 Kit Token 契约，因此本风格可与仓库内其它风格（cute / pixel / enterprise…）无痛互换——只换 tokens，组件代码不动。
