# 装饰艺术 / Art Deco

> 象牙奶油为底、深祖母绿为骨、金线为饰、墨为字的 1920 年代装饰艺术设计系统。几何对称的太阳放射、人字纹、阶梯切角与细双金线，配衬线大标题与宽字距大写小标。给复古奢华品牌——精品酒店、爵士俱乐部、剧院、香水屋、高级餐饮与活动——让"盖茨比式的华丽"现代、克制、对称，而非俗气的镀金堆砌。

示例主体「**The Verdance**」为原创虚构实体（1920 年代大饭店与爵士夜场），仅作展示，无任何第三方设计。

---

## 1. 一句话气质

把 1920 年代的几何华丽——大厅天花上的金色太阳放射、檐口的人字纹、菜单上的细双金线——翻译成一套能在屏幕上长期使用的现代界面语言：**象牙的温度、祖母绿的沉稳、金的点睛、墨的端正**。它该像一枚切割精良的胸针——华贵在那儿，却处处对称、处处有分寸。

---

## 2. 设计哲学 / 性格

本 Kit 继承统一的 `--ds-*` Token 契约（变量名、文件结构、组件清单全部一致），在此之上"换肤"出装饰艺术气质。它继承的是工程骨架，改写的是色、字、形、节奏四件事。

1. **象牙而非纯白。** 主背景是带暖意的象牙奶油（`#f6f1e7`），文字是带绿的深墨（`#1a2420`）。屏幕不再是冷白，而是有温度的奶油纸。
2. **祖母绿是骨，金是饰。** 深祖母绿（`#0f5a44`）承担主操作、品牌与选中态；金（`#8a6816`）只用于装饰线条、太阳放射 / 人字纹的描边、阶梯角与链接。绿做事，金点睛。
3. **几何即装饰，对称即纪律。** 太阳放射、扇形光、人字 / 锯齿、阶梯切角、双金线——全部是规整的几何，并保持强烈的垂直对称。装饰再多也得对齐、对称、成系统，而非随意堆砌。
4. **锋利方正，而非圆润。** 圆角近乎为零（`sm 0` / `md 2` / `lg 3`），徽章与按钮都是方正的——Deco 的端正来自直角与切角，不来自大圆角。
5. **衬线作骨、宽字距大写作饰、黑体作肉。** 大标题走 Noto Serif 衬线带文气，小标 / 眉题 / 字号标签走 Inter 宽字距大写（letter-spacing ~0.2em）带招牌字感，正文走无衬线保证可读。三者的反差本身即层级。

**与基底的关系：** 继承 Geist 的变量契约与组件结构；改掉它的中性灰与冷白——这套风格的身份恰恰在于象牙、祖母绿、金线与几何装饰。

**与 11-luxury 的区别（重要）：** luxury 是极简炭灰 + 拉丝金、暗色优先、大量留白、几乎无纹样；Art Deco 则是**装饰性、对称、祖母绿为主、亮色优先、金线密度更高**——一个是"少即是多"的奢侈，一个是"几何华丽"的奢侈。

## 3. 何时用 / 何时别用

**适合**
- 复古奢华品牌：精品酒店、爵士 / 鸡尾酒俱乐部、剧院、歌剧院、晚宴俱乐部。
- 香水、高级珠宝、烈酒、巧克力等"华丽几何"调性的产品。
- 婚礼 / 庆典 / 颁奖 / 周年活动页与高端票务。
- 复古主题的内容站、菜单、节目单、邀请函。

**不适合**
- 高密度数据 / 强工具感的 B 端后台（用企业 / B-side）。
- 强科技 / 极客气质的开发者产品（用暗黑科技）。
- 面向儿童的活泼场景（用可爱风）。
- 想要极简、品牌无装饰倾向的通用系统（用 Swiss / luxury）。

> 提醒：别把"装饰艺术"做成满屏镀金、处处发光。本风格的高级感来自**对称的克制**——大量象牙留白 + 祖母绿主色 + 一缕金线装饰，金线要细、要成系统、要对齐。

| Token | 亮 | 暗 | 用途 |
| --- | --- | --- | --- |
| `--ds-primary` | `#0f5a44` | `#d8b25a` | 主操作 / 选中（亮=祖母绿 ≥6.2:1，暗=金 ≥9.0:1） |
| `--ds-primary-hover` | `#0c4a38` | `#e3c172` | hover |
| `--ds-primary-active` | `#093829` | `#c49e44` | active / pressed |
| `--ds-primary-fg` | `#f6f1e7` | `#0d1714` | primary 上的文字 |
| `--ds-accent` | `#8a6816` | `#5fb89a` | 金（链接 / 次强调 / 饰线，亮 ≥4.6:1）；暗为绿玉 |
| `--ds-accent-fg` | `#f6f1e7` | `#06140f` | accent 上的文字 |

**强调色梯度（亮 · 祖母绿）：** `primary-active 093829` → `primary 0f5a44` → `primary-hover 0c4a38`。
按下更深、悬停略深、默认居中——形成自然的"按压感"。

> **暗色 primary 是「金」配深字。** 暗色主操作色为金 `#d8b25a`，其上文字为深绿黑 `#0d1714`（`--ds-primary-fg`）——**这是有意为之**：金底配浅字过不了 AA，金底配深字才达标（≥9.0:1）。金按钮在夜场模式下用深字而非白字。

> **金的纪律：** 金（`--ds-accent`）在亮色作链接、次强调与**装饰线条**（太阳放射 / 人字纹 / 阶梯角 / 双金线的描边）。亮色金 `#8a6816` 对象牙底达 4.6:1，可作小字 / 链接；但**不要**用它写大段正文。装饰用的金线要细（1–2px），要对称成系统。

## 5. 字体

**对比轴：衬线大标题 + 宽字距大写小标 + 无衬线正文。** 衬线带来 Deco 招牌的优雅笔锋，宽字距大写带来 1920 年代金属字母的仪式感，无衬线保证屏幕可读，三者的反差本身即层级。

- `--ds-font-serif`：`"Noto Serif", Georgia, "Songti SC", serif`
  优雅的衬线大标题；拉丁字形走本地 Noto Serif，中文走 Songti / 系统宋回退——预览页不外链任何中文字体。
- `--ds-font-sans`：`"Inter", -apple-system, "PingFang SC", sans-serif`（正文 / 控件；**眉题 / 标签 / 字号标签用大写 + letter-spacing ≈0.16–0.22em** 作 Deco 招牌字感）。
- `--ds-font-mono`：`"JetBrains Mono", monospace`（数据 / 代码 / tabular 数字）。

**比例尺**（标题拉开、正文舒适）：

| Token | px | 典型用途 |
| --- | --- | --- |
| `--ds-text-3xl` | 52 | 首屏大标题（衬线） |
| `--ds-text-2xl` | 38 | h1 |
| `--ds-text-xl` | 28 | h2 |
| `--ds-text-lg` | 22 | h3 / 小标题 |
| `--ds-text-md` | 18 | 导语 |
| `--ds-text-base` | 16 | 正文 |
| `--ds-text-sm` | 14 | 元信息 / 控件 |
| `--ds-text-xs` | 12 | 脚注 / 宽字距大写标签 |

**字重：** normal 400 / medium 500 / **bold 700**（衬线标题用 700；宽字距大写小标用 700 + uppercase）。
**字距：** 衬线标题 `--ds-tracking-tight: -0.01em`（略收，避免粘字）；正文 0；**大写小标另加 0.16–0.22em**。
**行高：** tight 1.22（标题）/ normal 1.55 / relaxed 1.8（长正文）。

**中文适配：** 标题宋体配 -0.01em，正文黑体行高 1.8、行宽锁 ≤75ch。宽字距大写处理对中文不适用——中文标签用正常字距即可，仅拉丁字形做 uppercase + tracking。

## 6. 间距与布局

- **比例尺**（4px 基）：1=4 / 2=8 / 3=12 / 4=16 / 5=20 / 6=24 / 7=32 / 8=40 / 9=48 / 10=64 / 11=80 / 12=96。
- **节奏：** 组内 8–12、组间 16–24、区块层级 48–80。Deco 讲究对称留白——区块之间敢留白，靠双金线与几何饰带分层。
- **对称：** 优先**强垂直对称**：太阳放射居中、阶梯角四角呼应、人字带全宽对齐。装饰元素成对、居中、对齐网格。
- **容器宽：** 内容站约 1120px；长文阅读容器锁 ~680px（≤75ch）。
- **栅格：** 12 栏意识，常用 2 / 3 / 4 列卡片网格，≤640px 收为单列。

---

## 7. 圆角 / 阴影 / 描边

- **圆角（锋利方正）：** `sm 0` / `md 2` / `lg 3` / `full 9999`。Deco 的端正来自直角与**切角 / 阶梯角**（用 `clip-path` 做多边形切角徽章），而非大圆角。
- **阴影（克制）：** 三档极轻投影（`md`/`lg` 仅一层柔影），靠双金线、阶梯角与对称分层，而非堆叠投影。**不做** 1px 描边 + 大模糊阴影并存的 ghost-card。
- **描边：** 常规 `--ds-border`、强描边 `--ds-border-strong`；点睛处用金（`--ds-accent`）画**细双金线**（`3px double` 或 `inset box-shadow` 双线）作分隔与精选卡内框。装饰线条全部细而对称。

---

## 8. 动效

- **时长：** fast 140ms / base 220ms / slow 320ms。
- **曲线：** `--ds-ease: cubic-bezier(.22,.61,.36,1)`——温润 ease-out，无弹跳，配 Deco 的优雅节制。
- **典型过渡：** 按钮底色 / 描边渐变、卡片 hover 抬升、浮层淡入上移、主题切换全页色彩流动。
- **`prefers-reduced-motion`：** 预览页与组件均在 `@media (prefers-reduced-motion: reduce)` 下将动画 / 过渡降到 0.01ms，并关闭平滑滚动；任何金光闪烁 / 装饰动效都须遵循此设置。

## 9. 组件规格

控件文字走无衬线，大标题走衬线，小标 / 标签走宽字距大写；颜色一律 `var(--ds-*)`。

- **按钮（全状态）**
  - 高度：小 32 / 中 40 / 大 48；圆角 `--ds-radius-sm`(0)；字号 sm/base。
  - 主操作：祖母绿底 `--ds-primary` + `--ds-primary-fg`，hover→`-hover`、active→`-active`（暗色主操作为金底深字）。
  - 次操作：`--ds-bg-elevated` 底 + `--ds-border-strong` 边 + `--ds-fg`。
  - 文字按钮：透明底 + 祖母绿字。危险：`--ds-danger` 底 + 白字。
  - 禁用：`opacity:.5` + `not-allowed`；焦点：`--ds-focus-ring`（祖母绿 / 金 双环）。
- **输入框 / 表单**：高 40、圆角 0、`--ds-bg-elevated` 底 + `--ds-border` 边；focus 换 `--ds-primary` 边 + 焦点环；错误态 `--ds-danger` 边 + 提示字；含 textarea / select / checkbox / radio / switch（`accent-color: --ds-primary`）。
- **卡片**：`--ds-bg-elevated` + 1px `--ds-border` + `--ds-radius-md` + `--ds-shadow-sm`；hover 升至 `--ds-shadow-md` 并强描边。**精选卡用细双金线内框**（`inset box-shadow` 双线点缀）。
- **徽章 / 标签**：方正（`--ds-radius-sm`），高 22，**宽字距大写**；默认中性，另有 primary / success / warning / danger / info；金线描边标 `.gold` 作 Black Tie / Est. 1924 等装饰标。
- **提示条 Alert**：**前置语义 SVG 图标 + 浅色 `*-bg` 底 + 1px 全描边**（无侧边色条）；四语义齐全。
- **导航**：顶栏（sticky，半透明象牙底 + 1px 底分隔 + 金切角徽 + 宽字距大写品牌字）+ Tabs（下划线选中走祖母绿）+ 面包屑 + 分页。
- **装饰组件**：太阳放射 / 扇形光（hero 居中背景）、人字 / 锯齿金带（章节饰带）、阶梯切角（四角呼应）、细双金线（分隔）、切角金钻徽（领标 / 角标）——全部内联 SVG / `clip-path`，对称且走 `var(--ds-accent)`。
- **其他**：下拉菜单、工具提示、模态框、Toast、步进器——层级走语义 `--ds-z-*`（dropdown<sticky<modal-backdrop<modal<toast<tooltip）。

## 10. 可访问性

- **对比度：** 正文 ≥4.5:1，大字 / UI ≥3:1，亮暗双主题均已校验达标（含弱文字 `--ds-fg-subtle` 与象牙底上的金 accent）。
- **金按钮的暗色文字：** 暗色 primary 为金（`#d8b25a`），其上文字为深绿黑（`#0d1714`，≥9.0:1）——金底用深字才达 AA，不可改用浅字。
- **金不作大段正文：** 装饰用金（`--ds-accent`）只作链接 / 次强调 / 细线，不写长正文（亮色金 4.6:1 适合小字 / 链接 / 大字，不适合密排正文）。
- **焦点：** 所有可交互元素 `:focus-visible` 显示 `--ds-focus-ring`（祖母绿 / 金 双环），键盘可见。
- **触控目标：** 控件最小高度 ≥40px，移动端可点区域 ≥44px。
- **可减弱动效：** 遵循 `prefers-reduced-motion`，金光 / 闪烁降为静态。
- **语义图标：** 状态 / 章节 / logo / 全部装饰纹样全部内联 SVG（零 emoji），图标带 `aria-hidden` 或图标按钮带 `aria-label`。

## 11. Do / Don't

**Do**
- 大量象牙留白 + 祖母绿主色 + 一缕金线装饰——对称的克制出高级。
- 装饰几何要**对称、对齐、成系统**：太阳放射居中、阶梯角四角呼应、人字带全宽对齐。
- 大标题用衬线，小标 / 眉题用宽字距大写，正文用黑体，保持三段对比轴。
- 金只作细描边、太阳放射 / 人字纹线条、阶梯角、双金线与链接。
- 圆角近零、用切角 / 阶梯角做端正感；精选卡用细双金线内框。

**Don't**
- 别满屏镀金、处处发光、金线乱堆——华丽不等于多，俗气是本风格头号翻车点。
- 别用金（`--ds-accent`）写大段正文或铺大色块（亮色金过不了密排正文 AA，且显俗）。
- 别把装饰做得不对称 / 不对齐——Deco 的灵魂是几何对称，乱了就垮。
- 别用大圆角 / 重投影 / 渐变文字冲淡端正；别忽略象牙底上对比（正文须 ≥4.5:1）。
- 别和 11-luxury 撞脸：那是暗色极简拉丝金，本风格是亮色装饰祖母绿——保持装饰性与对称。
- 别用 emoji 当图标 / 装饰——一律内联 SVG。

## 12. 如何接入

1. **引入 Token（必做）：** 在入口引 `tokens.css`，即获得 `:root` 亮色 + `[data-theme="dark"]`（含 `.dark` 兼容）暗色。
   ```html
   <link rel="stylesheet" href="./styles/21-art-deco/tokens.css">
   ```
2. **用变量，不硬编码：** 组件一律 `var(--ds-*)`，例如：
   ```css
   .btn-primary { background: var(--ds-primary); color: var(--ds-primary-fg); border-radius: var(--ds-radius-sm); }
   .deco-card   { background: var(--ds-bg-elevated); border: 1px solid var(--ds-border); box-shadow: var(--ds-shadow-sm); }
   .gold-rule   { border-bottom: 3px double var(--ds-accent); }      /* 细双金线分隔 */
   .featured    { box-shadow: inset 0 0 0 3px var(--ds-bg-elevated), inset 0 0 0 4px var(--ds-accent); } /* 双金线内框 */
   .eyebrow     { text-transform: uppercase; letter-spacing: .2em; color: var(--ds-accent); } /* 宽字距大写小标 */
   ```
3. **切换主题：** 设 `document.documentElement.dataset.theme = 'dark'`（或加 `.dark` 类）即切祖母绿之夜模式。
4. **Tailwind 项目：** 引预设并导入 tokens.css：
   ```js
   // tailwind.config.js
   module.exports = { presets: [require('./styles/21-art-deco/tailwind.preset.js')] }
   ```
   即可用 `bg-bg text-fg bg-primary text-primary-fg border-border rounded-sm font-serif` 等工具类，暗色自动跟随 `[data-theme="dark"]` / `.dark`。
5. **字体：** 拉丁字形可引共享 `_fonts/fonts.css`（本地 Noto Serif / Inter / JetBrains Mono）；中文走 Songti / 系统宋回退，无需外链。
